    Weekend: Python Variables and Modules Import

    Python Variables

    Variables are named references to objects.

    Here is an example of how we create a variable in Python in the Python shell:

    >>> x = 4
>>> x
4

    We create variables by using the assignment operator = . Variables do not store any value, but point to where the value is created in the computer's memory.

    When we equate two variables we are referencing the variables to the same value.

    Python variables can store values from any data type. Since Python is dynamic in nature we do not have to specify what type of data we are storing.

    >>> x = 12
>>> x
12
>>> x = "Hello"
>>> x
Hello
>>> pie = 3.1423
>>> pie
3.1423

    Python Modules

    A Python module is a single executable (can be run or launched) Python file.It contains code to do a particular task and has a .py extension.

    Let us create this on our desktops. Create a folder and name it MS-Python-Pre-work. Create a new file and name it module_test.py. Open Atom and type in the following:

    module_test.py

    greeting = "Hello"
school = "Moringa"

print(greeting + ", " + school + "!")

    To execute this code, navigate to the project folder in the console (where module_test.py is located) and run the following command:

    $ python3.6 module_test.py
Hello, Moringa!

    We type in python3.6 then the name of the file module_test.py.

    The code is immediately executed and we get our output as Hello, Moringa!

    A Python program is a collection of modules that are linked to perform a particular function.

    Import

    Python comes with a lot of pre-written code in for of modules and packages.

    1. A module is a single Python file that contains code to do a particular task.
    2. A package is made up of one or more modules that are joined to perform a particular task.

    Modules and packages belong to Python's standard library. They extend Python's functionality, and it is important to learn how to use them.

    import this

    $ python3.6 example.py

    math Module

    Let us look at a more useful module, named math. Replace the contents of example.py with the following code:

    example.py

    import math

x = math.sqrt(9)
print(x)

y = math.ceil(x)
print(y)

    Let's run it:

    $ python3.6 example.py
3.0
3

    Here, we import the math module. The math module comes with a lot of built-in methods, and we've used a couple of them here: the sqrt() method to find the square root of 9 (3.0), which we represent with the variable x; and the ceil() method to round up x (3).

    The syntax to use when we use module methods is module_name.method_name(arguments), where module_name is the name of the module (math in this case) and method_name is the name of the method (we've used sqrt() and ciel()).

    from Keyword

    You can also import a module method directly using the keyword from. Try this out in example.py:

    example.py

    from math import sqrt,ceil

x = sqrt(9)
print(x)

y = ceil(x)
print(y)

3.0
3

    Notice here that the syntax changes: we were able to use sqrt() and ciel() without typing math first. When we use the from keyword, we can use the method name directly without referencing the module.

    random Module

    Let us use another fun module called random. Replace the contents of example.py with the following code:

    example.py

    import random

# We can generate a random number using the randint method

random_number = random.randint(0,10)

print(random_number)

    Here used the import keyword, to get the random module. From the module we used its randint() method and passed in two parameters to define the range of values from which Python will choose a random number (in our example, 0 is the low end of the range and 10 is the high end). When we run the file, it returns a random number, from the range provided by the parameters.

    Now let's run it a few times:

    $ python3.6 example.py
4
$ python3.6 example.py
0
$ python3.6 example.py
7

    The output on your console may look different, but the thing to note here is that it's generating a different value each time we run example.py. We have the random module to thank for that.
