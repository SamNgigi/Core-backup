    Weekend: Input, sys.argv and Control Flow

    Python Variables

    Variables are named references to objects.

    Here is an example of how we create a variable in Python in the Python shell:

    >>> x = 4
>>> x
4

    We create variables by using the assignment operator = . Variables do not store any value, but point to where the value is created in the computer's memory.

    When we equate two variables we are referencing the variables to the same value.

    Python variables can store values from any data type. Since Python is dynamic in nature we do not have to specify what type of data we are storing.

    >>> x = 12
>>> x
12
>>> x = "Hello"
>>> x
Hello
>>> pie = 3.1423
>>> pie
3.1423

    Python Modules

    A Python module is a single executable (can be run or launched) Python file.It contains code to do a particular task and has a .py extension.

    Let us create this on our desktops. Create a folder and name it MS-Python-Pre-work. Create a new file and name it module_test.py. Open Atom and type in the following:

    module_test.py

    greeting = "Hello"
school = "Moringa"

print(greeting + ", " + school + "!")

    To execute this code, navigate to the project folder in the console (where module_test.py is located) and run the following command:

    $ python3.6 module_test.py
Hello, Moringa!

    We type in python3.6 then the name of the file module_test.py. The code is immediately executed and we get our output as Hello, Moringa!

    A Python program is a collection of modules that are linked to perform a particular function.

    math Module

    Let us look at a more useful module, named math. Replace the contents of example.py with the following code:

    example.py

    import math

x = math.sqrt(9)
print(x)

y = math.ceil(x)
print(y)

    Let's run it:

    $ python3.6 example.py
3.0
3

    Here, we import the math module. The math module comes with a lot of built-in methods, and we've used a couple of them here: the sqrt() method to find the square root of 9 (3.0), which we represent with the variable x; and the ceil() method to round up x (3).

    The syntax to use when we use module methods is module_name.method_name(arguments), where module_name is the name of the module (math in this case) and method_name is the name of the method (we've used sqrt() and ciel()).

    from Keyword

    You can also import a module method directly using the keyword from. Try this out in example.py:

    example.py

    from math import sqrt,ceil

x = sqrt(9)
print(x)

y = ceil(x)
print(y)

3.0
3

    Notice here that the syntax changes: we were able to use sqrt() and ciel() without typing math first. When we use the from keyword, we can use the method name directly without referencing the module.

    random Module

    Let us use another fun module called random. Replace the contents of example.py with the following code:

    example.py

    import random

# We can generate a random number using the randint method

random_number = random.randint(0,10)

print(random_number)

    Here used the import keyword, to get the random module. From the module we used its randint() method and passed in two parameters to define the range of values from which Python will choose a random number (in our example, 0 is the low end of the range and 10 is the high end). When we run the file, it returns a random number, from the range provided by the parameters.

    Now let's run it a few times:

    $ python3.6 example.py
4
$ python3.6 example.py
0
$ python3.6 example.py
7

    The output on your console may look different, but the thing to note here is that it's generating a different value each time we run example.py. We have the random module to thank for that.

    Input and sys.argv

    In our applications, we have basically been typing out values to everything in our code. Now let us explore a different way to obtain data into our application by configuring user input.

    input() function

    The input() function is called inside our Python application and prompts the user to pass in an input, which is stored as a string.

    Let us create a new folder in our MS-Python-Pre-work directory called Input, and create a new file input_demo.py and type in this:

    input_demo.py

    print("What is your name?")
name = input()

print("How old are you?")
age = int(input())

print(name)
print(age)

    We then run the file:

    $ python3.6 input_demo.py
"What is your name?"
James
"How old are you?"
19

James
19

    Here we are prompting the user for their name and age, and storing them in variables. Since age is a number, we used the int() function to convert it to an integer.

    Using the sys Module

    The sys module provides variables used or maintained by the interpreter. It has the list sys.argv, which contains the arguments passed into the command line when running the command to execute the program. Let's see how to use sys.argv to take input from the command line. Open up input_demo.py and replace the contents this code:

    input_demo.py

    import sys

name = sys.argv[1]

print("How old are you?")
age = int(input())

print(name)
print(age)

    Now let's run the file:

    $ python3.6 input_demo.py James
"How old are you?"
19

James
19

    Here we have first imported the sys module so that we could use sys.argv to store user input from the console and store it in a list. This is different from the previous example, where we used prompts along with input() to get the name of our user. Notice that we entered our name as the second argument of the command where we executed the program (the first argument was input_demo.py):

    $ python3.6 input_demo.py James

    Since sys.argv returns a list, we used sys.argv[1] in input_demo.py to retrieve this value - this simply tells our application to pick the argument at index 1 when our application is run and store it as variable name.

    We can pass multiple arguments when we run our application:

    input_demo.py

    import sys

name = sys.argv[1]
age = sys.argv[2]

print(name)
print(age)

    We then run the file with the necessary arguments:

    $ python3.6 input_demo.py James 19
James
19

    Here we pick both the name and age from the command line arguments passed.

    Control Flow

    Up to now our applications haven't been that interesting. Let us add some logic to our applications.

    if statements

    An if statement runs only when the condition passed to it evaluates to true. Let's experiment with this in input_demo.py.

    input_demo.py

    height = 74 # The unit is inches
if height > 70 :
    print("You are really tall")

    Run the file in the console:

    $ python3.6 input_demo.py
You are really tall

    Since height is above 70, the if statement will be called and it will print out the statement. This syntax differs from other programming languages like JavaScript which use curly braces ({} )to hold blocks of code, such as this example:

    var height = 74
if (height > 70) {
  console.log("You are really tall")
}

    Python uses indents. Indents are just four spaces we give to our application to define blocks of code. If we don't indent blocks of our code, we get an IndentationError and our program won't run.

    Comparison Operators

    Let us look at some of the comparison operators we will be using in Python

    • > Greater than
    • < Less than
    • ==Equal to
    • !=not equal to
    • >= Greater than or equal to
    • <=Less than or Equal to
    • and Checks if both conditions evaluate to true
    • or Checks if at least one condition is true
    • not returns the opposite of the condition given

    else

    The else statement comes right at the end of the if statement. It is run only when the if statement evaluates to False. Let's add an else statement to our example program:

    input_example.py

    height = 54 # inches
if height > 70 :
    print("You are really tall")
else:
    print("You are really short")

    Now when we run the file, we get a different message:

    $ python3.6 input_demo.py
You are really short

    The height is not greater than 70 so the if statement evaluates to False and the else statement is called.

    elif

    What if we have more than one condition to check for? We can use elif, which will allow us to check for multiple conditions. Looking at our example:

    height = 68 # inches
if height > 70 :
    print("You are really tall")
elif height > 60:
    print("You are of average height")

else:
    print("You are really short")

    Now let's run it:

    $ python3.6 input_demo.py
You are of average height

    Since the height is greater than 60, the elif statement will be executed first and Python never reaches the else statement.

    Checking for nothing

    In Python an empty value is automatically considered to be False. Consider this code:

    input_example.py

    name = ""
list_a = []

if list_a:
    print("I will not run")
else:
        print("I am Empty")

    When we run this code, nothing prints to the console.

    $ python3.6 input_demo.py
I am Empty

    This is because the if statement only runs if the output evaluates to be True, and since our list is empty the condition evaluates to False
